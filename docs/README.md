# TCP Chat

Консольный TCP‑чат на Go с поддержкой комнат и сохранением сообщений в SQLite.

---

## Описание

Проект представляет собой простое клиент‑серверное чат‑приложение:

- сервер принимает TCP‑подключения, управляет пользователями и комнатами, сохраняет сообщения в базу данных;
- консольный клиент подключается к серверу и предоставляет текстовый интерфейс с командами (список комнат, смена никнейма, помощь и т.п.);
- протокол обмена инкапсулирован в пакете `protocol`, работа с БД — в пакете `storage`.

Стек:

- **Go** (модуль `tcpChat`);
- **TCP**‑соединения из стандартной библиотеки;
- **SQLite** через `modernc.org/sqlite`;
- логирование через собственный пакет `logger`.

---

## Структура проекта

- `app/main.go` — точка входа, выбор режима `server` или `client`;
- `app/server/*` — логика TCP‑сервера, приём подключений, рассылка сообщений, обработка команд;
- `app/client/*` — консольный клиент, UI, чтение/запись в сеть;
- `app/storage/*` — инициализация БД, таблицы для комнат и сообщений;
- `app/protocol/*` — типы и кодек протокола;
- `app/config/config.go` — чтение конфигурации из переменных окружения;
- `.env.example` — пример настроек;
- `go.mod` / `go.sum` — зависимости.

---

## Требования

- Go **1.21+** (или совместимая версия, указанная в `go.mod`);
- SQLite (используется встраиваемый драйвер `modernc.org/sqlite`, отдельный сервер не нужен).

---

## Конфигурация

Используются переменные окружения:

- `CHAT_ADDR` — адрес, на котором слушает сервер и к которому подключается клиент (по умолчанию `:9000`);
- `CHAT_DB_PATH` — путь к файлу базы данных SQLite (по умолчанию `chat.db`).

Обычно достаточно создать `.env` в корне проекта:

```env
CHAT_ADDR=:9000
CHAT_DB_PATH=chat.db
```

и загрузить его перед запуском (через `direnv`, `docker-compose` или вручную в оболочке).

---

## Запуск сервера и клиента

Из корня репозитория:

```bash
# запуск сервера
go run ./app server

# запуск клиента (можно несколько раз в разных консолях)
go run ./app client
```

Сервер поднимет TCP‑листенер на `CHAT_ADDR`, инициализирует/создаст SQLite‑базу (`CHAT_DB_PATH`) и начнёт принимать подключения.

Клиент подключится к тому же адресу и откроет консольный интерфейс. Базовая подсказка доступна командой:

```text
/help
```

## Основные возможности

- подключение нескольких клиентов к одному серверу;
- обмен сообщениями в реальном времени;
- логика чата вынесена в отдельные пакеты (удобно расширять);
- хранение истории сообщений и информации о комнатах в SQLite;
- конфигурация через переменные окружения.
